Cloud costs permanently reduced

High cloud costs are often the result of poor architecture decisions at the beginning of a software project. Infrastructure costs grow faster than generated revenue, thereby destroying the business model. Here are three concrete examples of massive savings.

-€150k cloud costs / year
Google Cloud Platform
Microsoft Azure

GCP: Save €25k/year or 55% with 10 lines of code

In a Google Cloud project, inefficient programming logic forgot to cache failed database queries, allowing millions of read accesses to always pass through to the database and causing immense costs. Only 10 lines of code for temporary storage (= caching) helped solve this problem and reduced database costs by 55% per month or €25k per year.

GCP: Save €75k/year with Serverless Architecture

A customer software used classic cloud instances (virtual machines) to run its application. These ran 24/7 and were designed for peak loads. By switching to serverless approaches, costs adapt to the actual network traffic of your application – meaning low traffic = lower costs! By automatically scaling to zero during inactivity, even more costs can be saved. Through the points listed below, it was possible to save an additional €75k per year in a customer project.

  • Pay-per-request instead of flat-rate rental with Google Cloud Run
  • Shutdown of test environments through simple automation
  • Use of spot instances for batch jobs with Dataflow

Microsoft Azure: Save €4k/month with correct log data settings

Frequently, the costs for monitoring and the associated collection of log data are underestimated. Too often, all log data from an application is simply collected indiscriminately and stored in a so-called Log Analytics Workspace from Azure. Here, the configuration was not adapted to the actual needs of the business. A 3-minute reconfiguration of the existing Azure cloud resources resulted in savings of approximately €4k per month.
